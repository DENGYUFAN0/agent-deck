# METHODOLOGY.md — 用 HTML 取代 PPT 做组会汇报的方法论

一句话：让**任何一家 AI agent** 按同一份契约，生成「单文件、可放映、可现场编辑、可一键导 PDF」的 HTML 幻灯片。**HTML 是唯一源文件，PDF 是定稿快照。**

---

## 1. 为什么 HTML 能改变 PPT 汇报的格局

| 维度 | PPT (.pptx) | agent-deck (.html) |
|------|-------------|--------------------|
| AI agent 修改 | 二进制格式，须借助专门工具链，迭代慢、易丢样式 | 纯文本，任何 agent 整文件读写，一轮对话即改完 |
| 会中修改 | 要退出放映 → 改 → 再放映 | 按 E 当场改，导师意见直接写在片子上 |
| 版本管理 | 靠文件名自觉，改了什么无从查起 | 版本记录内嵌文件内（谁的哪版改了什么），文件名自动带版本号 |
| 分发归档 | 对方没装 Office / 字体缺失就走样 | PDF 快照由浏览器打印引擎渲染（同机所见即所得）；HTML 双击即开 |
| 依赖 | 装 Office | 一个浏览器（Chrome/Edge），断网可用 |

**边界诚实**（什么时候仍然用 PPT）：
- 需要复杂动画、切换效果、嵌入视频音频 → PPT 更顺手；
- 学校/会议**明确要求交 .pptx** → 用 pptx 流程，别硬转；
- 需要多人同时协作编辑 → 用 Google Slides / 腾讯文档；
- agent-deck 的编辑是"单机单人"模型，这是刻意的简化（零依赖换来的）。

## 2. 两条核心能力的技术定案（以及为什么这么定）

### 2.1 前端修改 + 记录

**定案：contenteditable + 三层持久化。**

```
第 1 层  内存        —— 正在编辑的 DOM
第 2 层  localStorage —— 400ms 防抖自动暂存（防误关浏览器，安全网）
第 3 层  自序列化落盘 —— 「保存新版本」把整个页面 outerHTML 下载成新 .html 文件
                        （唯一可信存档，版本号 +1，修改说明写入内嵌 JSON 记录）
```

为什么这么定：
- **contenteditable 而非自建编辑器**：浏览器原生、零依赖，任何 agent 都写得出来；代价是编辑体验是"改文字"级别而非"拖版式"级别——组会场景够用。
- **localStorage 只当安全网、不当存档**：它绑定浏览器+源（origin），换浏览器、换电脑、隐私模式就没了。**文件才是真身**——这条是本方法论最重要的纪律。
- **自序列化（self-serializing HTML）是"记录"能力的关键**：修改后的内容、递增的版本号、带修改说明的日志，全部固化在新文件里。文件本身携带自己的历史，不依赖任何外部系统。

### 2.2 一键导出 PDF

**定案：`window.print()` + 严格打印样式。**

- `@page { size: 1280px 720px; margin: 0 }` —— 1280×720px @96dpi 恰好等于 PowerPoint 16:9 页面（13.333×7.5 英寸），导出的 PDF 和 PPT 导的观感一致；
- 每张幻灯片 `page-break-after: always`，最后一张设 `auto`（防末尾空白页）；
- `print-color-adjust: exact` 保住背景色；打印时隐藏一切放映 UI、取消缩放变换。

为什么不用 html2pdf.js / jsPDF：
1. 要引外部库 → 违反零依赖（断网不可用、文件寿命受制于 CDN）；
2. 中文字体嵌入是大坑；
3. 渲染是"模拟"的，保真度必然低于浏览器自己的打印引擎。

"一键"的实情：一键唤起打印对话框 + 手点一次「保存」。两次点击，换来零依赖和打印引擎级的渲染保真——注意保真是"同机保真"：字体来自系统字体栈，定稿 PDF 要在自己常用的机器上导。

**进阶（真·零点击，可选）**：Windows 自带 Edge 支持命令行无头导出：

```powershell
& "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --headless --print-to-pdf="D:\out.pdf" "file:///D:/path/deck.html"
```

首次使用先核对一次输出的页面尺寸是否正确（无头模式对 @page 的支持随版本变化），确认无误后可存成脚本。日常用打印对话框即可，不必依赖此路径。

## 3. 硬契约（跨 agent 的接口标准）

完整条文见 [PROMPT.md](PROMPT.md) 第 A 段，这里列骨架。**契约即接口**：任何 agent 生成的文件只要守约，就能被其他 agent 继续修改、被同一套流程编辑/存版本/导 PDF。

1. **单文件、零依赖、离线可用**（禁 CDN、禁网络字体；图表用内联 SVG）
2. **16:9 / 1280×720 设计稿**，键盘翻页、全屏、页码、进度条、#页码深链
3. **E 键编辑模式**：contenteditable + 粘贴转纯文本 + localStorage 防抖暂存 + 重开提示恢复
4. **版本机制**：meta `deck-id` / `deck-version` + 内嵌 `revision-log` JSON + 「保存新版本」自序列化下载 `标题_vNN_日期.html`
5. **打印样式**：@page 尺寸=幻灯片尺寸、一片一页、隐藏 UI、保留背景色、无末尾空白页
6. **window.deck API**（go/next/prev/toggleEdit/saveVersion/exportPDF/serialize/version）

## 4. 组会工作流

```
会前  把大纲/素材 + PROMPT.md 发给 agent（能传文件就附上 template.html）
      → 得到 v1 → 跑 60 秒验收清单 → 预讲
会中  投屏 HTML 放映（F 全屏）
      导师意见 → 按 E 当场改在片子上（尤其"问题与讨论"页）
会后  「保存新版本」→ 填修改说明（例：罗老师建议扩大样本；调整第 4 章顺序）→ 得 v2
      「导出 PDF」→ 发群 / 归档
下次  把 v2.html 发给 agent：「在此基础上生成下次组会的汇报，本周进展是……」
```

版本文件天然形成一条研究进展的时间线：`选题_v01_0301.html → 选题_v05_0412.html → …`，每个文件内嵌自己的修改记录，任何时候双击任何一个版本都能完整放映。

## 5. 跨 agent 的三种用法

| 用法 | 场景 | 怎么做 |
|------|------|--------|
| **A. 纯提示词** | 任何聊天型 agent，不能传文件 | 粘贴 PROMPT.md 的 A 段 + 你的大纲 |
| **B. 模板参照** | agent 能收文件（保真度最高） | 发 template.html + 「参照此实现，替换为以下内容」 |
| **C. 增量修改** | 日常主用：改现有版本 | 发 vN.html + PROMPT.md 的 B 段 |

能力弱一些的模型容易在打印样式和序列化上偷工减料——所以验收清单不可省，不合格就把不合格条目原文丢回去让它修。

## 6. 已知坑与对策

| 坑 | 现象 | 对策 |
|----|------|------|
| localStorage 绑定浏览器 | 换电脑/换浏览器/隐私模式，暂存不见了 | 纪律：会后立刻「保存新版本」落盘；暂存只是安全网 |
| 打印没勾「背景图形」 | PDF 里色块全白 | 打印对话框勾选「背景图形」（模板已有 print-color-adjust 兜底，多数情况不勾也在） |
| 打印页眉页脚 | PDF 每页出现日期和文件路径 | 打印对话框关闭「页眉和页脚」 |
| 末尾空白页 | PDF 比幻灯片多一页 | 契约已规定最后一张 break 设 auto；若仍出现，检查 body 是否有多余 margin |
| 贴位图截图 | 文件暴涨、图不可编辑 | 让 agent 用内联 SVG 重画；确需照片时压缩后 base64 内嵌 |
| agent 偷偷引 CDN | 有网时正常、会议室断网白屏 | 验收第 1 条必须断网测 |
| 内联 JS 里出现 script 结束标签字面量（哪怕在注释里） | 脚本被 HTML 解析器截断，翻页/编辑/导出全部失效 | 契约已写明禁止；症状是打开后按键无反应，把文件丢回 agent 排查脚本截断 |
| 保存新版本后继续在旧文件里改 | 两份文件版本混乱 | 保存后立刻关旧文件、改用新文件（模板保存时会提示） |
| 习惯性 Ctrl+S 用浏览器「另存网页」 | 存的是磁盘原文件，修改全丢还以为存好了 | 契约已规定拦截 Ctrl+S 转为「保存新版本」 |
| 会中往一页里塞太多内容 | 超出 720px 的部分放映和 PDF 都被无声裁掉 | 编辑模式实时红框+提示警告；内容多就拆页 |
| Firefox/Safari 打印 | @page 像素尺寸支持不稳 | 放映随意，**导 PDF 认准 Chrome/Edge** |
| 换一台机器导 PDF | 系统字体不同，换行/字形细微漂移 | 定稿 PDF 在自己常用机器上导；字体栈已尽量选各平台近似字体 |

## 7. 验收清单

见 [PROMPT.md](PROMPT.md) 文末表格（8 条，60 秒）。核心四问：断网能开吗？能编辑且刷新后能恢复吗？保存出的新文件带版本记录吗？PDF 页数尺寸对吗？

## 8. 可选扩展模块（按需让 agent 加，别一次全要）

- **演讲者备注**：每页藏一段备注，N 键切换显示，打印时排除；
- **总览模式**：O 键缩略图九宫格，点击跳转；
- **批注层**：会中记录与正文分离（批注块样式醒目，可一键清空）；
- **计时器**：角落显示已用时长，组会限时汇报用；
- **自动目录页**：从各页 h2 生成目录并带页内跳转；
- **深色主题**：整套 CSS 变量换肤（正式汇报建议保持浅色，投影仪友好）。

每加一个模块，都必须守住总契约：单文件、零依赖、不破坏既有能力。
