# PROMPT.md — 万能提示词（粘给任何 AI agent）

> **默认走 A 段「母版填充」**：把 `template.html`（黄金母版）发给任何一家 AI agent（Claude / ChatGPT / Gemini / DeepSeek / Kimi …）——机件由母版携带，模型只填内容，中小模型也能一次做对。
> 只有开发新母版时才用 A+ 段「从零实现」（仅限最强模型）。B 段改现有文件（日常主用），C 段从旧 PPT 迁移。
> 生成后按文末「验收清单」自测 60 秒，不合格就把不合格的条目原文丢回给 agent 让它修。
> 也可以机器验收：`cd checker && npm install && node check.mjs 你的文件.html`（仓库自带合规校验器，逐条自动跑清单）。

## A. 母版填充（默认主模式 —— 任何模型都能干）

把 `template.html`（母版）连同下面这段话一起发给 agent；不能传文件的聊天框，就把母版全文粘贴在提示词后面。

```text
请以随附的 agent-deck 母版为基础，为我生成一份新汇报。母版自带全部机件（放映 / 现场编辑 / 版本存档 / 导出 PDF），你只负责内容，不负责机件。

【两条铁律】
1. 只允许改动母版中「✂ CONTENT-START ✂」与「✂ CONTENT-END ✂」两道界标之间的内容，外加 <title> 的文字和 meta deck-id 的 content 值（改成 主题英文或拼音-日期）。界标之外——<style>、<script>、工具栏、各浮层、meta deck-version——一个字符都不许动，也不许"顺手优化"。
2. 界标之内只使用下方组件说明书里的结构：不发明新类名、不写任何 <script>、不引入任何外部资源；图表用内联 SVG 并给图形元素加 data-label / data-value 属性（柱顶的静态数值必须保留，那是 PDF 的兜底呈现）。

【组件说明书】（每个组件母版里都有现成实例，照抄结构换内容）
- 普通页骨架：<section class="slide"> 内容… <div class="slide-foot"><span>题目缩写</span><span class="foot-num"></span></div> </section>
- 封面页：<section class="slide cover">：.kicker + h1 + .subtitle + .meta + aside.notes + .cover-hint
- 章节头：<p class="kicker">01 · 章节名</p> + <h2>标题</h2>
- 两栏：<div class="cols"><div class="col">…</div><div class="col">…</div></div>
- 重点结论框：<div class="box">…</div>；提问框：<div class="box ask"><b>1 · 类别</b>　问题 + 我的倾向</div>
- 状态签：<span class="tag done">完成</span>（另有 tag doing / tag risk）
- 表格：<table><tr><th>…</th></tr><tr><td>…</td></tr></table>
- 路线图：<ol class="roadmap"><li class="now"><b>③ 阶段名</b>说明</li>…</ol>（当前阶段加 now）
- 图表：内联 SVG（参照母版第 5 页），矩形带 data-label/data-value，柱顶印静态数值
- 演讲者备注：<aside class="notes">只有演讲者视图（S 键）可见</aside>
- 页数不限，但每页信息克制、一页一个重点；写不下就拆页（失败模式 F3）

【内容】
- 场景：【组会汇报 / 论文答辩 / 项目评审 / 课程展示 / 任意】
- 大纲与素材：【粘贴在此；没给全的地方按场景惯例补【】占位符，别空着也别编造】

【交付前自查】
- 界标之外是否零改动（对照母版逐段确认 <style> 与 <script> 原封未动）；
- 对照文末「失败模式目录 F1–F9」扫一遍（母版填充模式下重点防 F1 外部依赖、F3 超高）；
- 返回完整的单文件 HTML，不要只给片段。
```

## A+. 完整契约 · 从零实现运行时（进阶：只在开发新母版时用）

> **警告**：本模式要求模型从散文契约实现约 600 行运行时机件（舞台几何、序列化、打印、保存……），只有最强的模型能一次做对。实测记录：某中档模型走此模式，机件逻辑完成九成，却因舞台居中几何一处写错导致每一页错位裁切、整份报废。**日常生成汇报请用 A 段母版填充**；下面的完整契约同时也是所有模式的验收标准。

```text
请为我生成一个单文件 HTML 幻灯片（场景：【组会汇报 / 论文答辩 / 项目评审 / 课程展示 / 任意汇报场景】），严格满足以下契约。契约即接口：任何一条不满足都算失败。

【一、硬性要求 —— 缺一不可】

1. 单文件、零依赖、离线可用
   - 所有 CSS / JS 全部内联在这一个 .html 文件里；
   - 禁止引用任何 CDN、外部脚本、外部样式、网络字体（会议室可能没网，文件要能长期保存）；
   - 双击用 Chrome/Edge 打开（file:// 协议）即可放映；
   - 图片一律用内联 SVG 手绘或 base64 内嵌，图表必须用内联 SVG 画（可编辑、文件小），不要位图截图。

2. 幻灯片规格
   - 16:9，设计稿 1280×720 像素，自动缩放适配任意窗口大小；
   - 键盘 ←/→/空格/PageUp/PageDown 翻页，Home/End 跳首末页，F 全屏，Esc 关浮层；
   - 右下角页码「当前/总数」，底部细进度条；
   - 支持 URL 井号深链（#5 = 打开时直达第 5 页）；
   - 编辑状态下（焦点在可编辑区）翻页快捷键必须失效，否则打字会翻页。

3. 前端编辑（能力一）
   - 快捷键 E 或工具栏「编辑」按钮切换编辑模式；
   - 编辑模式下每张幻灯片整体 contenteditable，点击文字直接修改，有明显的视觉提示（如边框高亮 + 左上角状态角标）；
   - 粘贴自动转纯文本：拦截 paste 事件，用 Selection/Range 标准 API 插入纯文本（换行转 br 元素），不依赖已废弃的 document.execCommand（仅可作后备路径）；注意手动插入不会触发 input 事件，插入后需向所在幻灯片派发一次 input 以驱动自动暂存；
   - 所有修改自动暂存到 localStorage（400ms 防抖），key 格式为 "agent-deck:" + deck-id + ":v" + 版本号；
   - 重新打开文件时若发现暂存与当前内容不一致，顶部横幅提示「恢复修改 / 丢弃」；
   - 编辑模式下实时检测当前页内容是否超出幻灯片高度（超出部分在放映和 PDF 中都会被裁掉），超高时给出明显警告（红色边框 + 提示），退出编辑和序列化时清除警告标记。

4. 修改记录与保存（能力二·记录）
   - <head> 里有两个 meta：deck-id（全局唯一，用 主题+日期 生成）和 deck-version（初始为 1）；
   - <body> 末尾有 <script type="application/json" id="revision-log">[]</script> 作为内嵌版本记录；
   - 工具栏「保存新版本」按钮：弹窗让我填一句修改说明 → 版本号 +1 → 修改说明和时间追加进 revision-log → 把整个页面（含修改后的内容和更新后的记录）序列化为 '<!DOCTYPE html>\n' + document.documentElement.outerHTML，保存为新文件，文件名 = 标题_v版本号两位数_YYYYMMDD.html；
   - 保存路径首选 File System Access API（showSaveFilePicker 系统保存对话框，可直接写回原文件夹）；用户取消保存时必须回滚版本号与刚追加的记录；API 不存在或出错时降级为 Blob 下载；
   - 序列化前必须退出编辑模式、收起所有浮层，保证下载的文件是干净的；
   - 写入 revision-log 时把 JSON 字符串里的小于号替换为 Unicode 转义序列（反斜杠+u003c），防止修改说明里出现 script 结束标签、截断文档；
   - 特别注意：整个内联脚本里（包括 JS 注释、字符串字面量）都绝不能出现「</」紧跟「script」的连写字面量——HTML 解析器会在那里直接截断脚本，页面所有功能失效；
   - 工具栏「版本」按钮可查看全部版本记录（版本号、时间、说明），注意对说明文字做 HTML 转义再渲染；
   - 保存成功后清掉旧版本号的 localStorage 暂存；
   - 拦截 Ctrl/Cmd+S 调用「保存新版本」——浏览器原生「另存网页」保存的是磁盘上的原文件，会让用户以为存好了、实际修改全丢，这是必须堵死的陷阱。

5. 一键导出 PDF（能力二·导出）
   - 工具栏「导出 PDF」按钮：先退出编辑模式、收起浮层，再调用 window.print()；
   - 打印样式必须做到：@page { size: 1280px 720px; margin: 0; }（恰好等于 PPT 16:9 页面）；每张幻灯片恰好占一页（page-break-after: always，最后一张设为 auto 防止末尾空白页）；隐藏所有工具栏、页码、进度条、浮层、提示文字；所有元素加 -webkit-print-color-adjust: exact 和 print-color-adjust: exact 保留背景色；#deck 容器取消缩放变换（transform: none !important）和定位，让幻灯片按文档流排列；
   - 点按钮时用非阻塞提示告知我：打印对话框里目标选「另存为 PDF」、勾选「背景图形」、关闭「页眉和页脚」；
   - 同时监听 window 的 beforeprint 事件（用户绕过按钮直接 Ctrl+P 或从菜单打印时，也要自动退出编辑模式、收起所有浮层再打印）。

6. 放映 UI
   - 底部居中悬浮胶囊形工具栏：◀ ▶ | 编辑 | 保存新版本 | 版本 | 导出 PDF | 全屏 | ?；
   - 放映隐蔽性：工具栏平时完全隐藏（opacity 0 且 pointer-events none），鼠标移动时浮现、静止约 2.5 秒后淡出，编辑模式下常显；静止时连鼠标光标一并隐藏（cursor:none），放映中观众看不到任何操作痕迹；打开文件后先显示工具栏几秒帮助发现，封面和帮助浮层写明「移动鼠标唤出工具栏」；
   - 「?」帮助浮层：列出快捷键、编辑与保存的说明、导出 PDF 的打印设置三件事；
   - 暴露 window.deck 对象（go/next/prev/toggleEdit/saveVersion/exportPDF/serialize/version），方便日后自动化。

7. 机器验收锚点（供自动化校验，缺一即无法机检）
   - 固定 id：toolbar（工具栏）、counter（页码）、restore（恢复横幅）、revision-log（版本记录）；
   - 固定类名：幻灯片 .slide、当前页 .slide.active、编辑态 body.editing、超高警告 .slide.overflow；
   - 内容界标：「✂ CONTENT-START ✂」与「✂ CONTENT-END ✂」注释包住全部幻灯片（母版填充模式的机械边界）；
   - meta 名（deck-id / deck-version）、localStorage 前缀（agent-deck:）、window.deck API 签名均如上文；
   - 守住这些锚点，本仓库的合规校验器（checker/check.mjs）就能对任何 deck 自动跑完验收清单。

8. 演讲者备注与演讲者视图（可选增强：推荐实现，不实现不计验收失败）
   - 每页可含 <aside class="notes"> 备注块：放映与打印一律隐藏；编辑模式下显示为醒目备注块、可直接编辑，进入编辑模式时为没有备注的页自动补一个空备注块；
   - S 键或工具栏「演讲者」打开演讲者视图弹窗：本页备注、下一页预览、页码、计时；弹窗 DOM 全部由主窗口驱动，弹窗内不写任何 script（防 F2 脚本截断）；
   - 超高检测须临时排除备注块避免误报；序列化时清除空备注块保持存档干净。

9. L2 表达力模块（全部可选，按场景选配；实现任何一个都必须先过三问：断网单文件成立 / PDF 降级为可预期的静态呈现 / 编辑模式不坏）
   - 交互式图表（推荐首选）：SVG 图形元素带 data-label / data-value 属性；放映态鼠标悬停显示「标签：值」浮签（复用一个 fixed 定位的 tooltip 元素，id=charttip）；编辑模式自动停用悬停；打印隐藏 tooltip，且图上必须印有静态数值兜底（PDF 的降级呈现）；
   - 其余候选（章节地图 / 渐进显示 / 时间预算提醒 / 总览模式 / 批注层）见 METHODOLOGY §8 的模块目录与三问答卷。

【二、设计要求】

- 学术风格，浅色底：白底 #ffffff，正文 #1a2332，主色 #0f4c81（深蓝），辅色 #1a7f8e（青），浅灰线 #dde4ec，浅灰底 #f4f7fa；克制、干净，无花哨动画；
- 字体栈：-apple-system, "Segoe UI", "Microsoft YaHei", "PingFang SC", "Noto Sans CJK SC", "Malgun Gothic", sans-serif（不引外部字体，兼容中英韩）；
- 正文字号 ≥ 22px（1280×720 设计稿下），投影可读；每页信息密度克制，一页一个重点；
- 视觉资产形式优先级：CSS / 内联 SVG 矢量 → 几 KB 级小位图 base64 → 照片是最后手段（压缩后 base64）——资产形式必须与单文件约束兼容；
- 章节页眉格式统一：小号字距拉开的 kicker（如「01 · 进展总览」）+ 大标题 + 主色短下划线。

【三、内容】

按下面大纲生成幻灯片（下面是学术组会的预设大纲，其他场景直接替换成你的大纲即可；我没给全的地方按场景惯例补占位符，用【】标出让我填）：

- 封面：题目【……】、汇报人【……】、导师【……】、日期【……】
- 本周进展总览（上周承诺完成情况 + 一句话总结）
- 【……在这里粘贴你的大纲/素材……】
- 问题与讨论（列出想请导师给意见的事项，每项附上"我的倾向"）
- 下周计划（每项写可验证的完成标准）

【四、生成后自查】

逐条核对上面第一部分的 1–7，并在回复末尾输出一份自查表（每条打钩或说明偏差）。再对照「失败模式目录（F1–F9）」扫一遍。不要省略任何一条能力来"简化实现"。
```

## 验收清单（拿到文件后 60 秒自测）

| # | 动作 | 通过标准 |
|---|------|----------|
| 1 | 断网状态下双击打开 | 正常显示，无白屏、无报错 |
| 2 | 按 → 和 空格 | 正常翻页，右下角页码变化 |
| 3 | 按 E，点击标题打字 | 能直接修改，翻页键此时不生效 |
| 4 | 改完刷新页面 | 顶部出现「恢复修改 / 丢弃」横幅 |
| 5 | 点「保存新版本」填说明 | 下载出 `标题_v02_日期.html`，打开它点「版本」能看到刚才的说明 |
| 6 | 点「导出 PDF」→ 另存为 PDF | 页数 = 幻灯片数，每页完整 16:9、无裁切、无多余空白页、背景色在 |
| 7 | 打开 PDF | 无工具栏/页码等 UI 残留 |
| 8 | 用地址栏加 `#3` 打开 | 直达第 3 页 |

任何一条不过 → 把该条原文 + 现象描述发回给 agent：「验收第 X 条不通过，现象是……，请修复后返回完整文件」。

## 失败模式目录（F1–F9）

跨 agent 的共同语言：报障时直接说编号（例：「你犯了 F2，修掉再交」）。生成方 agent 交付前应对照本目录自查。

| 编号 | 失败模式 | 症状 | 修法 |
|------|----------|------|------|
| F1 | 外部依赖渗入 | 有网正常、断网白屏或字体错乱 | 删 CDN / 网络字体；图表改内联 SVG；确需照片压缩后 base64 |
| F2 | 脚本截断 | 打开后所有按键、按钮全部无反应 | 内联 JS 任何位置（含注释、字符串）不得出现「</」紧跟「script」的连写；写入 JSON 前把小于号转义 |
| F3 | 内容超高被裁 | 放映或 PDF 底部内容消失 | 拆页；实现编辑态超高红框警告 |
| F4 | 末尾空白页 | PDF 比幻灯片多一页 | 最后一张 .slide 的分页符设为 auto |
| F5 | 打印残留 UI | PDF 里出现工具栏 / 页码 / 浮层 | 打印样式隐藏全部放映 UI；监听 beforeprint 清场 |
| F6 | 编辑态漏进存档 | 保存出的新文件带编辑高亮或 contenteditable | 序列化前退出编辑模式、收起浮层、清掉状态类 |
| F7 | 翻页键劫持打字 | 编辑时一打字就翻页 | 键盘处理先判 isContentEditable，编辑焦点内翻页键失效 |
| F8 | 机检锚点漂移 | checker 大面积报错、无法自动验收 | 保住契约第 7 条的固定 id / 类名 / meta / 前缀 / API 签名 |
| F9 | 版本机制破坏 | 「保存新版本」报错或版本记录丢失 | 不改 revision-log 结构；版本号只增不减；取消保存须回滚 |
