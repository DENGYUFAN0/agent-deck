# PROMPT.md — 万能提示词（粘给任何 AI agent）

> 用法：复制下面代码块里的整段文字，发给任何一家 AI agent（Claude / ChatGPT / Gemini / DeepSeek / Kimi …），把【】里的内容换成你的。
> 如果对方 agent 能接收文件，把 `template.html` 一起发给它并加一句「参照此文件的实现方式」，保真度最高。
> 生成后按文末「验收清单」自测 60 秒，不合格就把不合格的条目原文丢回给 agent 让它修。
> 也可以机器验收：`cd checker && npm install && node check.mjs 你的文件.html`（仓库自带合规校验器，逐条自动跑清单）。

## A. 从零生成一份新汇报（主提示词）

```text
请为我生成一个单文件 HTML 幻灯片（场景：【组会汇报 / 论文答辩 / 项目评审 / 课程展示 / 任意汇报场景】），严格满足以下契约。契约即接口：任何一条不满足都算失败。

【一、硬性要求 —— 缺一不可】

1. 单文件、零依赖、离线可用
   - 所有 CSS / JS 全部内联在这一个 .html 文件里；
   - 禁止引用任何 CDN、外部脚本、外部样式、网络字体（会议室可能没网，文件要能长期保存）；
   - 双击用 Chrome/Edge 打开（file:// 协议）即可放映；
   - 图片一律用内联 SVG 手绘或 base64 内嵌，图表必须用内联 SVG 画（可编辑、文件小），不要位图截图。

2. 幻灯片规格
   - 16:9，设计稿 1280×720 像素，自动缩放适配任意窗口大小；
   - 键盘 ←/→/空格/PageUp/PageDown 翻页，Home/End 跳首末页，F 全屏，Esc 关浮层；
   - 右下角页码「当前/总数」，底部细进度条；
   - 支持 URL 井号深链（#5 = 打开时直达第 5 页）；
   - 编辑状态下（焦点在可编辑区）翻页快捷键必须失效，否则打字会翻页。

3. 前端编辑（能力一）
   - 快捷键 E 或工具栏「编辑」按钮切换编辑模式；
   - 编辑模式下每张幻灯片整体 contenteditable，点击文字直接修改，有明显的视觉提示（如边框高亮 + 左上角状态角标）；
   - 粘贴自动转纯文本：拦截 paste 事件，用 Selection/Range 标准 API 插入纯文本（换行转 br 元素），不依赖已废弃的 document.execCommand（仅可作后备路径）；注意手动插入不会触发 input 事件，插入后需向所在幻灯片派发一次 input 以驱动自动暂存；
   - 所有修改自动暂存到 localStorage（400ms 防抖），key 格式为 "agent-deck:" + deck-id + ":v" + 版本号；
   - 重新打开文件时若发现暂存与当前内容不一致，顶部横幅提示「恢复修改 / 丢弃」；
   - 编辑模式下实时检测当前页内容是否超出幻灯片高度（超出部分在放映和 PDF 中都会被裁掉），超高时给出明显警告（红色边框 + 提示），退出编辑和序列化时清除警告标记。

4. 修改记录与保存（能力二·记录）
   - <head> 里有两个 meta：deck-id（全局唯一，用 主题+日期 生成）和 deck-version（初始为 1）；
   - <body> 末尾有 <script type="application/json" id="revision-log">[]</script> 作为内嵌版本记录；
   - 工具栏「保存新版本」按钮：弹窗让我填一句修改说明 → 版本号 +1 → 修改说明和时间追加进 revision-log → 把整个页面（含修改后的内容和更新后的记录）序列化为 '<!DOCTYPE html>\n' + document.documentElement.outerHTML，用 Blob 下载为新文件，文件名 = 标题_v版本号两位数_YYYYMMDD.html；
   - 序列化前必须退出编辑模式、收起所有浮层，保证下载的文件是干净的；
   - 写入 revision-log 时把 JSON 字符串里的小于号替换为 Unicode 转义序列（反斜杠+u003c），防止修改说明里出现 script 结束标签、截断文档；
   - 特别注意：整个内联脚本里（包括 JS 注释、字符串字面量）都绝不能出现「</」紧跟「script」的连写字面量——HTML 解析器会在那里直接截断脚本，页面所有功能失效；
   - 工具栏「版本」按钮可查看全部版本记录（版本号、时间、说明），注意对说明文字做 HTML 转义再渲染；
   - 保存成功后清掉旧版本号的 localStorage 暂存；
   - 拦截 Ctrl/Cmd+S 调用「保存新版本」——浏览器原生「另存网页」保存的是磁盘上的原文件，会让用户以为存好了、实际修改全丢，这是必须堵死的陷阱。

5. 一键导出 PDF（能力二·导出）
   - 工具栏「导出 PDF」按钮：先退出编辑模式、收起浮层，再调用 window.print()；
   - 打印样式必须做到：@page { size: 1280px 720px; margin: 0; }（恰好等于 PPT 16:9 页面）；每张幻灯片恰好占一页（page-break-after: always，最后一张设为 auto 防止末尾空白页）；隐藏所有工具栏、页码、进度条、浮层、提示文字；所有元素加 -webkit-print-color-adjust: exact 和 print-color-adjust: exact 保留背景色；#deck 容器取消缩放变换（transform: none !important）和定位，让幻灯片按文档流排列；
   - 点按钮时用非阻塞提示告知我：打印对话框里目标选「另存为 PDF」、勾选「背景图形」、关闭「页眉和页脚」；
   - 同时监听 window 的 beforeprint 事件（用户绕过按钮直接 Ctrl+P 或从菜单打印时，也要自动退出编辑模式、收起所有浮层再打印）。

6. 放映 UI
   - 底部居中悬浮胶囊形工具栏：◀ ▶ | 编辑 | 保存新版本 | 版本 | 导出 PDF | 全屏 | ?；
   - 放映隐蔽性：工具栏平时完全隐藏（opacity 0 且 pointer-events none），鼠标移动时浮现、静止约 2.5 秒后淡出，编辑模式下常显；静止时连鼠标光标一并隐藏（cursor:none），放映中观众看不到任何操作痕迹；打开文件后先显示工具栏几秒帮助发现，封面和帮助浮层写明「移动鼠标唤出工具栏」；
   - 「?」帮助浮层：列出快捷键、编辑与保存的说明、导出 PDF 的打印设置三件事；
   - 暴露 window.deck 对象（go/next/prev/toggleEdit/saveVersion/exportPDF/serialize/version），方便日后自动化。

7. 机器验收锚点（供自动化校验，缺一即无法机检）
   - 固定 id：toolbar（工具栏）、counter（页码）、restore（恢复横幅）、revision-log（版本记录）；
   - 固定类名：幻灯片 .slide、当前页 .slide.active、编辑态 body.editing、超高警告 .slide.overflow；
   - meta 名（deck-id / deck-version）、localStorage 前缀（agent-deck:）、window.deck API 签名均如上文；
   - 守住这些锚点，本仓库的合规校验器（checker/check.mjs）就能对任何 deck 自动跑完验收清单。

【二、设计要求】

- 学术风格，浅色底：白底 #ffffff，正文 #1a2332，主色 #0f4c81（深蓝），辅色 #1a7f8e（青），浅灰线 #dde4ec，浅灰底 #f4f7fa；克制、干净，无花哨动画；
- 字体栈：-apple-system, "Segoe UI", "Microsoft YaHei", "PingFang SC", "Noto Sans CJK SC", "Malgun Gothic", sans-serif（不引外部字体，兼容中英韩）；
- 正文字号 ≥ 22px（1280×720 设计稿下），投影可读；每页信息密度克制，一页一个重点；
- 章节页眉格式统一：小号字距拉开的 kicker（如「01 · 进展总览」）+ 大标题 + 主色短下划线。

【三、内容】

按下面大纲生成幻灯片（下面是学术组会的预设大纲，其他场景直接替换成你的大纲即可；我没给全的地方按场景惯例补占位符，用【】标出让我填）：

- 封面：题目【……】、汇报人【……】、导师【……】、日期【……】
- 本周进展总览（上周承诺完成情况 + 一句话总结）
- 【……在这里粘贴你的大纲/素材……】
- 问题与讨论（列出想请导师给意见的事项，每项附上"我的倾向"）
- 下周计划（每项写可验证的完成标准）

【四、生成后自查】

逐条核对上面第一部分的 1–7，并在回复末尾输出一份自查表（每条打钩或说明偏差）。不要省略任何一条能力来"简化实现"。
```

## B. 增量修改现有文件（日常最常用）

把现有的 `xxx_v03_20260707.html` 整个文件发给 agent（或粘贴内容），加这段：

```text
这是我组会用的 agent-deck 格式 HTML 幻灯片（单文件零依赖、E 键前端编辑、内嵌版本记录、window.print 导出 PDF）。
请在保持全部既有能力和视觉风格不变的前提下，做以下修改：

1. 【例：在"核心结果"后新增一页"稳健性检验"，内容如下……】
2. 【例：把第 5 页的柱状图改成折线图，数据：……】

要求：
- 不得引入任何外部依赖；不得破坏 meta deck-id / deck-version / revision-log 结构；
- 新增页面必须带 .slide 结构和 slide-foot 页脚，图表用内联 SVG；
- 返回完整的单文件 HTML（不要只给片段）。
```

## 验收清单（拿到文件后 60 秒自测）

| # | 动作 | 通过标准 |
|---|------|----------|
| 1 | 断网状态下双击打开 | 正常显示，无白屏、无报错 |
| 2 | 按 → 和 空格 | 正常翻页，右下角页码变化 |
| 3 | 按 E，点击标题打字 | 能直接修改，翻页键此时不生效 |
| 4 | 改完刷新页面 | 顶部出现「恢复修改 / 丢弃」横幅 |
| 5 | 点「保存新版本」填说明 | 下载出 `标题_v02_日期.html`，打开它点「版本」能看到刚才的说明 |
| 6 | 点「导出 PDF」→ 另存为 PDF | 页数 = 幻灯片数，每页完整 16:9、无裁切、无多余空白页、背景色在 |
| 7 | 打开 PDF | 无工具栏/页码等 UI 残留 |
| 8 | 用地址栏加 `#3` 打开 | 直达第 3 页 |

任何一条不过 → 把该条原文 + 现象描述发回给 agent：「验收第 X 条不通过，现象是……，请修复后返回完整文件」。
