# Changelog

## v0.3.0 — 2026-07-08

「借思路不借功能」：从炫技型竞品提炼可迁移原理，立成正式准则并落地第一个模块。

### 准则（METHODOLOGY §8 重写）
- **能力分层**：L0 核心契约（机检必过）/ L1 通用增强（推荐可选）/ L2 表达力模块（场景选配）；
- **L2 三问**（缺一不进）：断网单文件成立？PDF 降级为可预期的静态呈现（informed loss, never silent loss）？编辑模式不坏？
- 模块目录建账：交互图表 ✅ / 章节地图 / 渐进显示 / 时间预算提醒 / 总览模式 / 批注层 / 深色主题；
- 设计要求新增资产形式优先级：CSS/SVG 矢量 → KB 级 base64 小图 → 照片最后手段。

### 新增（首个 L2 模块）
- **交互式图表**（契约第 9 条）：SVG 元素带 `data-label`/`data-value`，放映态悬停显示精确值；编辑模式自动停用；打印隐藏浮签、以图上静态数值兜底。

## v0.2.0 — 2026-07-08

定位升级：从「组会工具」到通用汇报范式。本轮吸收了 GitHub 同类项目考察的经验（TiddlyWiki 的保存层级、竞品三家共识的演讲者工具、ppt-agent-skill 的失败模式目录思路）。

### 新增
- **保存直写磁盘**：「保存新版本」首选 File System Access API（系统保存对话框，可直接写回原文件夹），用户取消时回滚版本号与记录；不支持的浏览器降级为下载（契约第 4 条更新）；
- **演讲者备注与演讲者视图**（契约第 8 条·可选增强）：每页 `aside.notes` 备注块（放映与 PDF 全隐、编辑模式直接写、自动补空块）；S 键弹出演讲者窗口——本页备注、下一页预览、页码、计时，主窗口与弹窗双向同步，弹窗内零 script；
- **失败模式目录 F1–F9**：跨 agent 的报障共同语言，入双语契约，生成方交付前对照自查。

### 校验器
- 新增第 28 项静态检查（保存实现须含 File System Access + 下载兜底）；超高检测排除备注块防误报；序列化清除空备注块。

## v0.1.0 — 2026-07-07

首个公开版本 / First public release。从方法论到基础设施的完整闭环。

### 核心能力
- 单文件零依赖 HTML 幻灯片模板（template.html）：16:9 放映、键盘导航、`#N` 深链、全屏；
- 现场编辑（E 键，contenteditable）+ localStorage 自动暂存 + 重开恢复横幅；
- 「保存新版本」自序列化落盘：版本号自增、修改说明与时间内嵌文件（revision-log）；
- 一键导出 PDF：window.print + @page 1280×720px（恰好等于 PPT 16:9 页面），一片一页、无空白页；
- 跨 agent 契约：PROMPT.md（中文正典）/ PROMPT.en.md（英文版），三种用法（从零生成 / 增量修改 / PPT 迁移）+ 60 秒验收清单。

### 鲁棒性
- 拦截 Ctrl+S 转「保存新版本」（浏览器原生另存会无声丢修改）；
- beforeprint 兜底：直接 Ctrl+P 也自动退出编辑、收起浮层；
- 编辑内容超高实时红框警告（超出部分放映与 PDF 均被裁掉）；
- 粘贴转纯文本走 Selection/Range 标准 API（已废弃的 execCommand 仅作后备）；
- 放映隐蔽：工具栏与鼠标光标静止 2.5s 自动隐藏、鼠标唤出、编辑常显。

### 基础设施
- 合规校验器 checker/（27 项自动验收：断网加载、翻页、编辑、暂存、序列化、打印样式、PDF 页数与 16:9 尺寸机检）；
- GitHub Actions CI：每次 push 对模板跑全套校验；
- 契约第 7 条「机器验收锚点」：任何守约 deck 皆可机检；
- 三语 README（English / 简体中文 / 한국어，中文为正典）+ Claude Code skill（skill/）。

### 已知限制
见 README「Known limits / 已知限制」：PDF 为同机保真、编辑是文字级非版式级、单人单文件（无实时协作）。
